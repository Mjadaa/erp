# نظام إدارة الموارد المتكامل للشركات الصغيرة (ERP System) 🚀

نظام ERP (Enterprise Resource Planning) متكامل ومجاني بالكامل، مصمم خصيصاً للشركات الصغيرة والمتوسطة كبرنامج سطح مكتب (Windows Desktop Application). يهدف النظام إلى أتمتة العمليات اليومية للشركة من محاسبة، ومخزون، ومبيعات، ومشتريات، وإدارة موارد بشرية، في بيئة واحدة متصلة ببعضها البعض ومؤمنة بالكامل.

---

## 🎯 الميزات الأساسية والوحدات (Modules)

1. **النظام المحاسبي (Accounting):** إدارة الحسابات، قيود اليومية، ميزان المراجعة، الأرباح والخسائر، والميزانية العمومية.
2. **إدارة المخزون (Inventory):** إدارة المنتجات، التصنيفات، المستودعات، الجرد، والتنبيهات بنفاد الكمية.
3. **إدارة الأصول (Assets):** تسجيل الأصول، الصيانة الدورية، وإهلاك الأصول.
4. **إدارة العملاء (CRM) والموردين (Suppliers):** سجلات مفصلة للعملاء والموردين، متابعة الديون، وأرشفة الفواتير.
5. **إدارة المبيعات والمشتريات (Sales & Purchases):** إصدار فواتير البيع والشراء وتأثيرها التلقائي على المخزون وحسابات الديون.
6. **إدارة الموظفين (HR):** بيانات الموظفين، الرواتب، وتتبع الحضور والانصراف.
7. **لوحة التحكم والتقارير (Dashboard):** مؤشرات أداء (KPIs) ورسوم بيانية تفاعلية تلخص حالة الشركة مالياً وإدارياً.

---

## 🔒 الأمان ونظام الصلاحيات (Security & Roles)

تم بناء النظام بأعلى معايير الأمان المتاحة لتطبيقات سطح المكتب لضمان عدم وصول أي شخص غير مخول لبيانات الشركة:

### 1. نظام الدخول الإجباري (Authentication)
* لا يمكن تشغيل أي جزء من النظام أو استعراض أي شاشة قبل المرور بشاشة تسجيل الدخول (Login).
* لا يوجد "تخطي" (Bypass)؛ كل طلب لقاعدة البيانات مقترن بهوية المستخدم.

### 2. حساب المدير الافتراضي (Default Admin)
* عند تشغيل النظام لأول مرة على جهاز جديد، يقوم النظام تلقائياً بإنشاء حساب المدير العام:
  * **اسم المستخدم:** `admin`
  * **كلمة المرور:** `admin123`
* يُجبر النظام المدير على تغيير كلمة المرور بعد أول عملية دخول لضمان حماية النظام.

### 3. الصلاحيات الديناميكية (Dynamic Roles & Permissions)
* لا يقتصر النظام على صلاحيات ثابتة، بل يمكن للمدير إنشاء "مجموعات صلاحيات" (Roles) غير محدودة، مثل: (محاسب، كاشير، مدير موارد بشرية).
* يتم تحديد مهام كل مجموعة بدقة متناهية (مثال: يمكن للمحاسب رؤية شاشة المحاسبة، ولا يمكنه رؤية شاشة رواتب الموظفين).
* **حماية الواجهات (UI Masking):** إذا لم يكن المستخدم يمتلك صلاحية لرؤية قسم معين (كالمشتريات مثلاً)، فإن الزر الخاص بهذا القسم يختفي تماماً من القائمة الجانبية (Sidebar).

## ⚙️ التقنيات المستخدمة (Technology Stack)

* **واجهة المستخدم (Frontend):** Flutter Desktop (Windows). يوفر أداءً فائقاً واستهلاكاً منخفضاً لموارد الجهاز مع واجهات عصرية جذابة.
* **قاعدة البيانات (Database):** SQLite (عبر حزمة `sqflite_common_ffi`). قاعدة بيانات محلية قوية جداً (تتحمل بيانات حتى 140 تيرابايت)، لا تحتاج لإنترنت ولا لخوادم (Zero-Configuration).
* **إدارة الحالة (State Management):** Provider.
* **دعم اللغات (Localization):** Easy Localization (يدعم العربية والإنجليزية بالكامل LTR/RTL).
* **الرسوم البيانية (Charts):** FL Chart لبناء رسومات الـ Dashboard.

## 🚀 كيفية تشغيل المشروع (How to Run)

إذا كنت تقوم بتشغيل المشروع لأول مرة في بيئة التطوير، اتبع الخطوات التالية:

1. **تحديث المكتبات:**
   ```bash
   flutter pub get
   ```

2. **تشغيل التطبيق على بيئة Windows Desktop:**
   *(تتطلب وجود Visual Studio Installer وتثبيت حزم Desktop development with C++)*
   ```bash
   flutter run -d windows
   ```

3. **التشغيل المؤقت على المتصفح (لتجربة الواجهات سريعاً):**
   ```bash
   flutter run -d chrome
   ```
